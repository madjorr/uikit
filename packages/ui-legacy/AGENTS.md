## What ships

- 40+ shadcn-style React components under `src/components/ui/`.
- 4 themes shipped as separate CSS exports: `acronis-default`,
  `acronis-ocean`, `cyber-chat`, `acronis-white-label`. White-label
  variants are generated by `scripts/generate-white-label-themes.ts`.
- A Tailwind preset (`./tailwind-preset`) so consumers can extend their
  own Tailwind config with the library's theme tokens.
- An auto-generated `llms.txt` produced at build time by
  `scripts/generate-llms-txt.ts` — a context file for AI agents working
  in **consumer** repos.

## Build pipeline

`pnpm build` runs three steps in order (don't run them out of order):

1. `build:lib` — `vite build -c vite.lib.config.ts`. The main library
   bundle and types (`vite-plugin-dts`).
2. `build:full-css` — `tsx scripts/build-full-css.ts`. Generates the
   per-bucket CSS bundles exposed via the `./styles/*` subpath exports.
3. `build:llms` — `tsx scripts/generate-llms-txt.ts`. Writes `llms.txt`
   alongside `dist/`.

## Stack specifics

- React 19 (catalog), TypeScript 5.9, Vite 6, Vitest 4, Storybook 10,
  Tailwind v4.
- **Primitives**: `@radix-ui/react-slot`, `@radix-ui/react-navigation-menu`
  as direct deps. **Base UI** (`@base-ui/react`) is a peer dep — the
  library uses it; consumers must install it.
- **Other peers**: `react`, `react-dom`, `next-themes`, `tw-animate-css`.
  All marked optional in `peerDependenciesMeta`.
- **zod 4** (intentional — `apps/demos` is still on zod 3; aligning is
  a tracked follow-up).
- **Color**: `culori` for theme generation utilities.

## Icons

Icons are generated, not authored:

- `scripts/generate-icons.ts` → component sources under
  `src/components/icons/`.
- `scripts/generate-icon-stories.ts` → matching Storybook stories.

Don't hand-edit files inside `src/components/icons/` — they get
overwritten on the next regen. Edit the generator scripts instead.

## When you change anything in `src/`

1. Add a Vitest test in `__tests__/` (see [`./context/testing.md`](./context/testing.md)).
2. Add a Storybook story in `__stories__/` covering all variants.
3. Add a Changeset: `pnpm changeset` (from repo root).
4. Verify it renders in `apps/demo` and, if it has a doc page, `apps/docs`.

See `../../context/releasing.md` for the Changesets / publish flow.
