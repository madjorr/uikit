# Contributing to `@acronis-platform/shadcn-uikit`

This is the **published** UI library. Bar for changes here is highest in
the monorepo: tests, stories, and a changeset are required.

See [the root CONTRIBUTING.md](../../CONTRIBUTING.md) for the
umbrella process (fork, branch, commits, PR etiquette). This file is
the workspace-specific layer.

For technical context (build pipeline, peer deps, file layout), see
[AGENTS.md](AGENTS.md).

## What requires a change here

- A new component or variant.
- A bug fix to an existing component.
- A change to the published exports map, peer deps, or build config.
- A new theme or token.

If your change is purely to a demo, a documentation page, or a
demo-only utility, it probably belongs in `apps/demos` or `apps/docs`
instead — those don't gate the release.

## Workflow

1. Implement the change in `src/components/ui/<component>/`.
2. Add or update a **Vitest** test in `src/components/ui/<component>/__tests__/<component>.test.tsx`
   covering render, key props, variants, and any interactive behavior.
3. Add or update a **Storybook** story in
   `src/components/ui/<component>/__stories__/<component>.stories.tsx` covering every variant, size,
   and state. The a11y panel should be clean.
4. If the component's prop types are too compound for `AutoTypeTable`
   to resolve in the docs site, add or update a `<component>.docs.ts`
   companion file (see [AGENTS.md](AGENTS.md) for when this applies).
5. Verify the change renders in both `apps/demo` (real-world flow) and,
   if a documentation page exists, `apps/docs`.
6. **Add a changeset** from the repo root: `pnpm changeset`. Pick the
   correct bump type (see below). Commit the generated file.

## Tests and verification

From the repo root (or `pnpm --filter @acronis-platform/shadcn-uikit ...`):

```bash
pnpm --filter @acronis-platform/shadcn-uikit test          # Vitest
pnpm --filter @acronis-platform/shadcn-uikit test:coverage # with v8 coverage
pnpm --filter @acronis-platform/shadcn-uikit storybook     # Storybook at :6006
pnpm --filter @acronis-platform/shadcn-uikit typecheck
pnpm --filter @acronis-platform/shadcn-uikit lint
```

Visual regression (Storybook + Playwright):

```bash
pnpm --filter @acronis-platform/shadcn-uikit storybook:test:visual

# Dockerized (deterministic, matches CI):
pnpm --filter @acronis-platform/shadcn-uikit storybook:test:visual:docker

# Update snapshots after an intentional visual change:
pnpm --filter @acronis-platform/shadcn-uikit storybook:test:visual:update
```

See [`./context/testing.md`](./context/testing.md) for deeper testing
guidance (happy-dom, localStorage polyfill, reliability rules).

## Changesets — bump type

- **patch** — internal refactor, bug fix, no API change.
- **minor** — new component, new variant, additive prop, new export.
- **major** — removed export, renamed prop, changed default behavior.
  Avoid until a major is planned.

If you're not sure, default to `patch` and ask in the PR.

## Files you should not hand-edit

- `src/components/icons/*` — regenerated by `scripts/generate-icons.ts`.
- `src/styles/themes/*-white-label.css` — regenerated by
  `scripts/generate-white-label-themes.ts`.
- `dist/`, `storybook-static/` — build artifacts.

Edit the generator scripts instead, then re-run them.

## Don't break the build pipeline

`pnpm build` runs three steps in order: `build:lib` → `build:full-css`
→ `build:llms`. If you change the exports map, vite config, or the
scripts, run the full `pnpm build` locally before pushing.
