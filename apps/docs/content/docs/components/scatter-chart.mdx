## Usage

```tsx
import { ScatterChart } from '@acronis-platform/ui-react';
import type { ChartConfig } from '@acronis-platform/ui-react';
```

`ScatterChart` is a typed composition over the shared [`Chart`](/components/chart)
primitives. Give it a `series` list (each `{ key, data }` with its own points), a
per-series `config`, and the numeric axis fields (`xKey` / `yKey`) ŌĆö it renders a
themed recharts scatter chart (tooltip, legend, axes, grid included), so you
don't hand-compose recharts children. Add a `zKey` to size points by a third
field (a bubble chart).

Each series carries its **own** point array (points aren't columns of a shared
row the way bar/line/area series are), so the API takes a `series` list rather
than `dataKeys`. Series colors are supplied by the caller through `config`, keyed
by each `series[].key`: `ScatterChart` turns those into the `--color-<key>`
custom properties its points fill from.

> **Design-pending v1**, ported from the demo `ScatterChartPlayground`. There is
> no chart token tier yet, so the example colors reference the shared brand/status
> tokens ŌĆö a dedicated `--ui-chart-*` data-viz palette is pending an upstream
> design pass. `--ui-background-status-strong-*` is chromatic in every brand;
> `--ui-background-brand-secondary` is brand-dependent, so it is not color-stable
> across brands until the real palette lands.

## Examples

  <ScatterChartDemo />
</DemoReact>

Two groups plotted on shared x/y axes, with a themed tooltip and legend:

```tsx
const series = [
  { key: 'classA', data: [{ hours: 2, score: 55 }, { hours: 6, score: 78 }] },
  { key: 'classB', data: [{ hours: 1, score: 70 }, { hours: 9, score: 95 }] },
];

const config = {
  classA: { label: 'Class A', color: 'var(--ui-background-brand-secondary)' },
  classB: { label: 'Class B', color: 'var(--ui-background-status-strong-danger)' },
} satisfies ChartConfig;

<ScatterChart
  config={config}
  series={series}
  xKey="hours"
  yKey="score"
  className="h-[360px] w-[520px]"
/>;
```

Turn it into a bubble chart by mapping a third numeric field to point size with
`zKey`:

```tsx
<ScatterChart config={config} series={series} xKey="hours" yKey="score" zKey="weight" zRange={[60, 500]} />
```

Change the marker with `shape`, and toggle the chrome with `showGrid` /
`showTooltip` / `showLegend`:

```tsx
<ScatterChart config={config} series={series} xKey="hours" yKey="score" shape="triangle" showLegend={false} />
```
