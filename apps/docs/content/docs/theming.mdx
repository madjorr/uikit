## How themes work

Acronis UIKit uses generated `--ui-*` CSS custom properties from
[`@acronis-platform/tokens-pd`](/packages/tokens-pd). You never hand-author
theme values — they are generated from the design system and resolved at paint
time, so brand and light/dark switches need no rebuild.

Load the default base, then (optionally) layer another brand override
on top. For full theming, import that brand's **bundled** entry
(`<brand>.all.css`) — it concatenates the semantic tier and every
component's token tier into one file, so a single import re-themes
every component ui-react ships:

```tsx
import '@acronis-platform/ui-react/styles'; // includes tokens-pd/css/default.css
import '@acronis-platform/tokens-pd/css/deep_sky_itkontoret.all.css'; // full brand override
```

### Partial theming

If you only want to override the shared semantic tier (colors most
components pull from) without touching per-component overrides, import
the semantic-only file instead — or reach for an individual
`<component>/<brand>.css` file to override just one component's tier:

```tsx
import '@acronis-platform/tokens-pd/css/deep_sky_itkontoret.css'; // semantic tier only
import '@acronis-platform/tokens-pd/css/Button/deep_sky_itkontoret.css'; // one component's tier only
```

## Dark mode

Dark mode is built into the tokens via `light-dark()` + `color-scheme`. Toggle
the active scheme with `data-theme`:

```html
<html data-theme="dark"></html>
```

## Theme switching

Brand switching is import-order based (last import wins), not a class toggle:

```css
@import '@acronis-platform/tokens-pd/css/default.css';
@import '@acronis-platform/tokens-pd/css/deep_sky_itkontoret.all.css';
```

## Design tokens

Key CSS variables you can override:

| Variable                    | Purpose                |
| --------------------------- | ---------------------- |
| `--ui-background-surface-primary`   | Main page background   |
| `--ui-background-surface-secondary` | Secondary surfaces     |
| `--ui-text-on-surface-primary`      | Primary text color     |
| `--ui-text-on-surface-secondary`    | Secondary / muted text |
| `--ui-border-on-surface-border`     | Dividers and borders   |
| `--ui-background-brand-primary`     | Brand accent color     |

## Custom theme

Override any variable in your CSS:

```css
:root {
  --ui-background-brand-primary: #7c3aed;
}
```
